# 파일업로드

1. 파일을 업로드한다.
   1-1. 파일종류는 엑셀파일로 제한한다.
2. 파일내용에 대한 검증은 1행의 컬럼명에 대한 비교확인으로 진행한다.
   2-1. 검증이 성공하면 파일을 저장소에 저장하고 경로 등 파일에 대한 메타정보와 파일 내용을 추출하여 JSON형태로 가공한 데이터를 저장한다. (file entity)
   2-2. 검증이 실패하면 파일업로드를 중지하고 오류를 응답한다.

# 파일내용 반영

1. 파일엔티티에 저장된 파일내용정보를 가져온다.
2. 적용하고자 하는 직원들을 선택한다.
3. 파일내용정보에서 선택된 직원들의 정보만 저장한다. (event-info entity, used-attendance entity)
   3-1. 파일내용 반영에 대한 이력을 저장한다. (file-content-reflection-history entity)

# 파일내용 반영 후 처리

1. event-info 와 used-attendance 를 가져온다.
2. holiday 정보를 가져온다.
3. 위 세가지 정보를 이용하여 선택된 연도-월-일에 대한 daily-event-summary를 생성한다.
   3-1. 이 과정에서 정상근무 범위를 벗어난 경우 attendance-issue 데이터를 생성한다.
   3-2. 정상근무 범위는 정책적으로 변동될 여지가 있기때문에 유연하게 구현한다.
4. 선택된 연도-월 에 대한 daily-event-summary 정보들을 가져와서 monthly-event-summary를 생성한다.
